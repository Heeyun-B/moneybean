# Django와 Vue를 활용한 금융상품 추천 및 커뮤니티 웹사이트 - 머니빈 (MoneyBean)



### 📋 프로젝트 개요

- 일시 : 2025년 SSAFY 1학기 최종 프로젝트
- 프로젝트명 : 머니빈 (MoneyBean)



### 📝 Description

- 금융 정보 및 자산 관리 서비스 제공 프로젝트
- 기획의도: 사용자 친화적인 UI/UX를 갖춘 종합 금융 플랫폼 제작 및 서비스 제공
  - 예적금, 금/은 시세 등 다양한 금융 상품 정보를 한 곳에서 제공
  - 사용자별 자산 관리 및 금융 학습 기능을 통한 실용적인 금융 서비스 구현
  - YouTube 검색 및 나중에 볼 영상 기능을 통한 금융 교육 콘텐츠 접근성 강화
  - **AI 기반 자산 분석 및 금융 뉴스 요약 서비스 제공**



### 🥇 GOAL

- 금융 데이터 기반 정보 제공 서비스 구성
- 사용자 맞춤형 자산 관리 기능 구현
- 커뮤니티 서비스 구성 (금융기사, 금융정보, 자유게시판)
- HTML, CSS, JavaScript, Vue.js, Django REST API, Database 등을 활용한 실제 서비스 설계
- 외부 API 연동 (OpenAI API, YouTube API, 금융감독원 API, Kakao Maps API, Naver News API)
- AI 활용을 통한 개인화된 금융 상품 추천 및 뉴스 요약 서비스 구현
- 서비스 관리 및 유지 보수

### 🤖 AI 금융상품 추천 알고리즘

사용자의 재정 상태를 다차원적으로 분석하여 최적의 금융 상품을 추천하는 AI 기반 알고리즘을 구현했습니다.

#### 1️⃣ 핵심 재정 지표 계산
4가지 핵심 지표를 통해 포트폴리오를 입체적으로 분석합니다.

1. 현금 비율 = (현금성 자산 / 총 자산) × 100

  적정 범위: 20-30% / 과다 보유 시: 적금 추천 / 부족 시: 단기 예금 추천

2. 투자 비율 = (투자 자산 / 총 자산) × 100

  연령별 적정 비율: 20-30대(70-80%), 40-50대(50-60%), 60대+(30-40%)

3. 부채 비율 = (총 부채 / 총 자산) × 100

  안정 구간: 30% 이하 / 주의 구간: 30-50% / 위험 구간: 50% 이상

4. 저축률 = (월 잉여금 / 월 소득) × 100

  건강한 저축률: 20% 이상

#### 2️⃣ 포트폴리오 균형 회복 전략
| 포트폴리오 상태 | 문제점 | 추천 전략 |
|-----------------|--------|-----------|
| 현금 과다 보유 (>40%) | 기회비용 발생 | 적금 상품 → 자산 효율화 |
| 투자 비중 과다 (>70%) | 위험 노출 과다 | 예금 상품 → 안정성 확보 |
| 부채 비율 높음 (>30%) | 재정 위험 증가 | 단기 고금리 예금 → 상환 준비금 마련 |
| 월 잉여금 충분 (>20%) | 저축 여력 높음 | 장기 적금 → 복리 효과 극대화 |

#### 3️⃣ 우선순위 결정 알고리즘
- **Priority 1 (최우선)**: 포트폴리오 균형 회복이 가장 시급한 상품 (4대 지표 중 기준치를 가장 크게 벗어난 항목 개선)
- **Priority 2 (추천)**: 수익률 최적화를 위한 보조 상품 (최고 금리 + 비대면 가입 편의성)
- **Priority 3 (고려)**: 장기 재무 목표 달성용 추가 상품 (장기 저축, 목돈 마련)

#### 4️⃣ AI 기반 개인화 (GPT-4o 활용)
GPT-4o를 활용하여 복잡한 재정 상황을 종합적으로 해석합니다.
- ✓ 4가지 핵심 지표를 종합적으로 해석
- ✓ 연령, 소득, 부채 등 다차원 변수 동시 고려
- ✓ 자연어로 추천 근거 설명 → 신뢰성 향상
- ✓ 프롬프트 수정만으로 추천 전략 유연하게 변경 가능

### 🤔 느낀점

기술적 성장: Vue 3 Composition API와 Pinia를 활용해 복잡한 전역 상태를 체계적으로 관리하고, 재사용 가능한 로직을 설계하는 아키텍처 역량을 키웠습니다.

UX 최적화: 전역 라우터 가드를 통한 리다이렉트 처리, 로딩 오버레이 등 사용자 관점의 세밀한 인터페이스 구현이 서비스 완성도에 미치는 영향을 체감했습니다.

데이터 관리 및 보안: 사용자별 localStorage 키 격리와 로그아웃 시 데이터 완전 초기화를 구현하며 클라이언트 단의 보안과 데이터 프라이버시의 중요성을 배웠습니다.

협업 및 API 통합: 다양한 외부 API(YouTube, 금감원 등)를 통합하고 백엔드와 긴밀히 소통하며, 효율적인 데이터 가공과 에러 핸들링 능력을 강화했습니다.


### 🎯 프로젝트 성과

- Vue 3 Composition API와 Pinia를 활용한 현대적인 프론트엔드 아키텍처 구현
