## 1. 프로젝트 개요

### 1.1. 서비스 목표 (Goal)
사용자가 금융 정보를 **'잘 알고'**, 자산을 **'잘 모으고'**, 소비를 <strong>'잘 쓰자'</strong>는 목표 아래, 개인의 금융 활동 전반을 지원하는 **종합 개인 자산 관리 플랫폼**을 구축합니다.

### 1.2. 주요 기능 정의 (Key Features)

| 구분 | 핵심 기능 | 상세 요구사항 |
| :--- | :--- | :--- |
| **잘 알고 (정보)** | 경제 뉴스 및 커뮤니티 | 주요 경제 뉴스 사이트에서 최신 기사 수집 및 목록 제공. 뉴스 기사에 대한 댓글 작성 및 의견 공유 기능. |
| **잘 모으고 (자산)** | 맞춤형 예적금 추천 | 사용자 정보 기반 최적의 예적금 상품 추천 및 시중 은행 금리 비교 기능 제공. |
| **잘 쓰자 (소비)** | 맞춤형 카드 추천 | 월 평균 소비 금액 및 소비 패턴 분석을 통한 혜택이 가장 큰 카드 추천. |
| **커뮤니티** | 할인 정보 게시판 | 사용자가 직접 할인 정보를 작성(Create), 조회(Read), 수정(Update), 삭제(Delete)하는 CRUD 게시판 운영. |
| **관리** | 프로필 대시보드 | 사용자의 활동 내역, 찜한 상품, 자산/소비 현황을 시각화하여 보여주는 기능. |

### 1.3. 홈 화면 구성 (Home Screen Mapping)
홈 화면은 서비스의 핵심 가치를 전달합니다.

* **똑똑한 자산관리의 시작**: 로그인/회원가입 섹션으로 사용자 인증을 유도합니다.
* **대시보드**: 로그인 후, '자산비중' 시각화와 '총자산 금액', '월지출 금액', '투자성향' 등 핵심 프로필 정보를 요약하여 보여줍니다.
* **바로가기**: '나에게 맞는 카드 찾아보기', '나에게 맞는 예/적금 찾아보기', 'News 목록', '게시판 목록' 등 주요 기능으로의 빠른 진입점을 제공합니다.

## 2. 기술 스택 (Technology Stack)

| 구분 | 기술 / 도구 | 비고 |
| :--- | :--- | :--- |
| **Frontend (FE)** | **Vue.js** | SPA(Single Page Application)로 구현. |
| **Backend (BE)** | **Django (DRF)** | RESTful API 서버로 구축. |
| **데이터 소스** | **금융감독원 API** | 금융 상품 데이터의 정확성 및 무결성 확보. |

## 3. 프로젝트 회고 (Retrospection)

### 🌟 느낀 점
* **사용자 경험(UX)의 중요성**: 금융 데이터는 복잡하지만, 홈 화면 대시보드와 같이 시각화를 통해 정보를 단순화하는 것이 사용자의 서비스 접근성을 크게 높임을 깨달았습니다.
* **통합적인 기획**: 프론트엔드와 백엔드 기능을 동시에 고려하여 API 명세 및 화면 설계를 진행하니, 개발 단계에서의 불필요한 시행착오를 줄일 수 있었습니다.

### 📚 배운 점
* **공공 API 연동 및 활용**: 금융감독원 API와 같은 외부 공공 데이터를 활용하여 신뢰할 수 있는 금융 상품 정보를 제공하는 방법을 배웠습니다.
* **RESTful API 설계**: Django DRF를 사용하여 기능 요구사항(CRUD)에 맞춰 효율적인 RESTful API 엔드포인트를 설계하는 경험을 했습니다.

### 🚧 어려웠던 점
* **추천 시스템 고도화**: 사용자의 금융 정보와 행동 데이터를 기반으로 최적화된 상품을 추천하는 로직을 설계하는 과정에서, 초기 데이터 축적 및 알고리즘 선택에 어려움이 있었습니다.
* **비기능 요구사항 충족**: 동시 접속성이나 응답 속도와 같은 비기능 요구사항을 만족시키기 위한 서버 및 DB 설계 최적화 방안을 모색하는 것이 도전적이었습니다.
